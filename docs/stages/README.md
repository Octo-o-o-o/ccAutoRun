## 📖 文档结构

每个阶段的详细文档包含以下内容：

- **导航链接**: 快速跳转到主计划、上一阶段或下一阶段
- **阶段信息**: 编号、预计时间、依赖关系、状态
- **目标**: 该阶段要达成的目标
- **任务清单**: 详细的待办任务列表
- **完成标准**: 验收标准
- **预期输出**: 该阶段产出的文件和功能
- **注意事项**: 重要提醒和最佳实践
- **验证命令**: 测试和验证的具体命令

## 🎯 使用指南

### 执行前
1. 阅读 [主执行计划](../../EXECUTION_PLAN.md) 了解整体规划
2. 查看当前阶段的详细文档
3. 确认所有依赖阶段已完成

### 执行中
1. 按照任务清单逐项完成
2. 运行验证命令确保质量
3. 更新文档中的完成状态

### 执行后
1. 检查完成标准是否全部满足
2. 运行验证命令确保功能正常
3. 创建 git commit
4. 更新主执行计划中的进度跟踪表
